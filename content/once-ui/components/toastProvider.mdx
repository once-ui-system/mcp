The `ToastProvider` wraps your app to provide access to `useToast`, allowing any component to trigger lightweight, dismissible toast messages globally.

## Basic usage

Wrap your app in `ToastProvider` to activate toast functionality.

<CodeBlock
  marginTop="16"
  marginBottom="24"
  codes={[
    {
      code:
`<ToastProvider>
    <App />
</ToastProvider>`,
      language: "tsx",
      label: "Setup"
    }
  ]}
/>

Use the `useToast()` hook to display notifications from anywhere in your app:

<CodeBlock
  marginTop="16"
  marginBottom="24"
  codes={[
    {
      code:
`const { addToast } = useToast();

addToast({
    variant: "success",
    message: "Saved successfully!",
});`,
      language: "tsx",
      label: "Trigger toast"
    }
  ]}
/>

## Action

Toasts can also include an optional `action` (e.g. a retry or undo button).

<CodeBlock
  marginTop="16"
  marginBottom="24"
  codes={[
    {
      code:
`addToast({
    variant: "danger",
    message: "Failed to save",
    action: <Button size="xs">Retry</Button>
});`,
      language: "tsx",
      label: "Action"
    }
  ]}
/>

## Toaster

The `ToastProvider` automatically renders a `<Toaster />` under the hood, passing it the current list of toasts and a `removeToast` handler.
