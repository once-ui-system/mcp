Once UI uses a config file to store app-level design configuration in a single place. You can change the fonts, colors, and more granular design preferences in the `app/resources/once-ui.config.js` file. This approach simplifies branding and help you maintain aeshetics between multiple Once UI apps.

## Set fonts

Once UI imports Geist and Geist Mono by default with [Next/font](https://nextjs.org/docs/pages/building-your-application/optimizing/fonts). To use a different font, simply change the imports and font declarations.

<CodeBlock
  marginTop="16"
  marginBottom="24"
  highlight="1-2,4,10"
  codes={[
    {
      code:
`import { Geist } from "next/font/google";
import { Geist_Mono } from "next/font/google";

const primaryFont = Geist({
    variable: "--font-primary",
    subsets: ["latin"],
    display: "swap",
});

const monoFont = Geist_Mono({
    variable: "--font-code",
    subsets: ["latin"],
    display: "swap",
});

const font = {
    primary: primaryFont,
    secondary: primaryFont,
    tertiary: primaryFont,
    code: monoFont,
}`,
      language: "tsx",
      label: "once-ui.config.js",
    },
  ]}
/>

## Style your app
        
Once UI uses data attributes to set color profiles, border radius style, themes, and other styling options. Data modes are inherited from the parent and can be applied to any
semantic HTML element, providing simple, seamless style overrides.

<CodeBlock
    marginTop="8"
    marginBottom="16"
    codes={[
  {
    code:
`theme:       'dark',         // dark | light
neutral:     'gray',         // sand | gray | slate
brand:       'blue',         // blue | indigo | violet | magenta | pink | red | orange | yellow | moss | green | emerald | aqua | cyan
accent:      'indigo',       // blue | indigo | violet | magenta | pink | red | orange | yellow | moss | green | emerald | aqua | cyan
solid:       'contrast',     // color | contrast
solidStyle:  'flat',         // flat | plastic
border:      'playful',      // rounded | playful | conservative
surface:     'translucent',  // filled | translucent
transition:  'all',          // all | micro | macro
scaling:     '100',          // 90 | 95 | 100 | 105 | 110`, 
    language: "tsx",
    label: "src/app/resources/config.js"
  }
]} />

## Override styles

Since data modes are inherited, it means you can nest them inside one-another, allowing
you to have a dark-mode app with a light-mode header for example, or use rounded buttons
in particular places in a generally sharp interface.

Just add the `data-theme`, `data-neutral`, `data-brand`, `data-accent`, `data-border`, `data-surface`, `data-solid`, `data-solid-style`, `data-transition` or `data-scaling` attributes to any semantic HTML element to override its default style.
